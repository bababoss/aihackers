📋 Overview

LTX-2 is a groundbreaking asymmetric dual-stream diffusion transformer that simultaneously generates synchronized video and audio content from text prompts. Unlike traditional sequential pipelines (text→video→audio), LTX-2 jointly models audio-video dependencies, enabling true temporal synchronization between modalities.

The architecture features:

14B Video Stream 5B Audio Stream 48 Transformer Blocks Bidirectional Cross-Attention

🚀 Generation Steps Explained

1

📝 Text Encoding

User prompt is processed through Gemma 3 text encoder, generating separate embeddings for video (4096-dim) and audio (2048-dim) conditions. This modality-specific encoding enables better phonetic accuracy for speech.

2

🎲 Latent Initialization

Random Gaussian noise is initialized in spatial format [B, C, F, H, W] (batch, channels, frames, height, width). For typical generation: [1, 128, 33, 16, 16].

3

📦 Patchification

Spatial latents are flattened to sequence format [B, seq_len, dim] to enable efficient transformer processing. This allows parallel processing across time, height, and width dimensions.

4

⏱️ Sigma Schedule

Noise schedule is generated targeting the computed sequence length. This controls the denoising trajectory from pure noise to coherent output over ~50 denoising steps.

5

🔄 Denoising Loop

For each step: create modality inputs with per-token timesteps and RoPE positions → forward through transformer (both conditional and unconditional for CFG) → apply guidance strategies → update latents using Euler solver.

6

⬅️ Unpatchification

Sequence format is converted back to spatial format [B, C, F, H, W], restoring the structured representation needed for VAE decoding.

7

🎬 VAE Decoding & Output

Video latents → Video VAE Decoder → Pixels (optionally upscaled). Audio latents → Audio VAE Decoder → Mel spectrogram → HiFi-GAN Vocoder → 24kHz audio waveform.

🔧 Core Components

🎬 Video VAE

Encoder: Compresses [B,3,F,H,W] → [B,128,F',H/32,W/32]
Decoder: Expands latents back to pixels
Compression: Frame: 1/8, Spatial: 1/32

🎵 Audio VAE

Encoder: Mel spectrograms → [B,8,T/4,16] latents
Decoder: Latents → Mel spectrograms
Temporal: 4× downsampling, ~1/25s per token

🔊 Vocoder

Type: HiFi-GAN based
Features: Stereo synthesis, modified for audio quality
Output: 24 kHz waveform from 16 kHz mel spectrograms

📝 Gemma Text Encoder

Backbone: Gemma 3-12B decoder-only LLM
Features: Multi-layer extraction, learnable fusion, separate V/A contexts
Output: 4096-dim video + 2048-dim audio embeddings

🎛️ Diffusion Components

Schedulers: LTX2Scheduler, LinearQuadratic
Guiders: CFG, STG (Spatio-Temporal), APG
Noisers: Add noise per schedule

⬆️ Spatial Upscaler

Purpose: Upsample latent representations for higher-resolution generation
Usage: Two-stage pipelines for better quality output

✨ Key Architectural Features

🎯 Asymmetric Dual-Stream Design

14B video stream captures spatiotemporal dynamics with 3D RoPE (x, y, t), while 5B audio stream processes 1D temporal sequences. Both share 48 transformer blocks but differ in width, optimizing for modality-specific information density.

🔗 Bidirectional Cross-Modal Attention

1D temporal RoPE enables sub-frame alignment between video and audio without spatial cross-attention, ensuring natural synchronization for lip-sync and foley effects. AdaLN gates provide diffusion timestep awareness for smooth transitions.

💾 Separate VAEs

Independent video/audio latent representations enable native V2A/A2V editing workflows and modality-specific compression optimization.

🌍 Multilingual Support

Gemma 3 backbone provides excellent multilingual understanding for phonetically accurate speech synthesis.

📊 Multi-Layer Features

Text encoder aggregates features from all Gemma layers, not just the final layer, for richer semantic understanding.

🎮 Fine-Grained Control

Perturbation system allows selective skipping of attention operations for advanced guidance techniques.

⚡ Quantization Support

FP8 quantization backends (TensorRT-LLM, FP8-Cast) reduce memory footprint while maintaining quality.

🔄 Cross-Modality AdaLN

Adaptive Layer Normalization gates conditioned on the other modality's hidden states for true synchronization.
