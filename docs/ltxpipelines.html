LTX-2 Pipelines

Production-Ready Audio-Video Generation Pipelines

📋 Overview

LTX-2 Pipelines provides production-ready implementations that abstract away the complexity of diffusion, model loading, and memory management. Each pipeline is optimized for specific use cases with different trade-offs between speed, quality, and memory usage.

Built using building blocks from ltx-core, these pipelines handle the complete inference flow including:

📦
Model Loading Automatic checkpoint handling
🔄
Encoding Text & image processing
🎬
Decoding VAE & vocoder output
💾
File I/O Video & audio export

🎯 Pipeline Selection Guide

Choose the right pipeline for your use case:

Do you need to condition on existing images/videos? ├─ YES → Do you have reference videos for video-to-video? │ ├─ YES → Use ICLoraPipeline (Video-to-Video) │ └─ NO → Do you have multiple keyframe images to interpolate? │ ├─ YES → Use KeyframeInterpolationPipeline (Animation) │ └─ NO → Use TI2VidTwoStagesPipeline (Image Conditioning) │ └─ NO → Text-to-video only ├─ Do you need best quality? │ └─ YES → Use TI2VidTwoStagesPipeline (Recommended) │ └─ Do you need fastest inference? └─ YES → Use DistilledPipeline (8 steps only)

📦 Available Pipelines

🎬 TI2VidTwoStagesPipeline

RECOMMENDED

Best for: High-quality text/image-to-video generation with upsampling.

Two-stage generation with Stage 1 generating low-resolution video with multimodal guidance, and Stage 2 upsampling to 2x resolution with distilled LoRA refinement. Supports image conditioning.

Use when: Production-quality output needed, higher resolution required, image-to-video, or quality over speed.

📺 TI2VidOneStagePipeline

EDUCATIONAL

Best for: Learning and quick prototyping only.

Single-stage generation without upsampling. Faster inference but lower resolution output (typically 512x768). Multimodal guidance and image conditioning supported.

Use when: Learning pipeline architecture, quick testing, or when high resolution is not critical.

⚡ DistilledPipeline

FASTEST

Best for: Fastest inference with good quality using distilled model.

Two-stage with only 8 predefined sigmas (8 steps in stage 1, 4 in stage 2). No guidance required. Fastest among all pipelines. Supports image conditioning.

Use when: Fastest inference priority, batch processing many videos, or only distilled checkpoint available.

🎥 ICLoraPipeline

VIDEO-CONTROL

Best for: Video-to-video and image-to-video transformations.

Two-stage with IC-LoRA support. Conditions on reference videos (video-to-video) or images at specific frames. CFG guidance in stage 1, upsampling in stage 2. Requires IC-LoRA trained model.

Use when: Video-to-video transformations, strong control needed, or reference videos available for guidance.

🎯 KeyframeInterpolationPipeline

ANIMATION

Best for: Generating videos by interpolating between keyframes.

Two-stage with keyframe interpolation using additive conditioning for smoother transitions. Multimodal guidance in stage 1, upsampling in stage 2.

Use when: Keyframe images need smooth transitions, animation/motion interpolation, or interactive frame-by-frame control needed.

🚀 How It Works - Complete Generation Pipeline

1

📝 Text Encoding with Gemma

User prompt is processed through Gemma 3 text encoder, generating separate context embeddings for video (4096-dim) and audio (2048-dim). This enables modality-specific semantic understanding for better phonetic accuracy.

2

🖼️ Optional Image/Video Processing

If conditioning on images or videos, they are encoded through VAEs to latent space. For image conditioning, latents are prepared for replacement or additive guidance. For video inputs, keyframes are extracted for IC-LoRA guidance.

3

🎲 Initialize Noise (Stage 1)

Random Gaussian noise is initialized for both video and audio in spatial format. Typical: [B, 128, 33, 16, 16] for video, [B, 8, 132, 16] for audio - representing low-resolution latents ready for denoising.

4

⏱️ Generate Sigma Schedule

Noise schedule is computed targeting the token count. For two-stage pipelines, Stage 1 typically uses 40 steps, Stage 2 uses 20 steps. Distilled pipelines use fixed 8+4 steps with predefined sigmas.

5

🔄 Denoising Loop with Multimodal Guidance

For each denoising step: (a) Add per-token timesteps and RoPE positions, (b) Forward through transformer both conditionally and unconditionally, (c) Apply multimodal guidance (CFG + STG + modality CFG), (d) Update latents using Euler solver.

6

⬆️ Stage 2: Upsampling with Distilled LoRA (if two-stage)

Denoised low-res latents from Stage 1 are fed into Stage 2. Distilled LoRA weights enhance quality while reducing computation. Another 20-step denoising loop produces high-resolution latents (2x upscaling).

7

🎬 VAE Decoding to Pixel/Audio Space

Final latents are decoded: Video latents → Video VAE Decoder → Pixels. Audio latents → Audio VAE Decoder → Mel spectrogram → HiFi-GAN Vocoder → 24kHz audio waveform. Both outputs are synchronized.

8

💾 Export Video and Audio

High-quality video frames and synchronized audio are combined and exported to MP4 file with specified frame rate (typically 25 fps for video, 24 kHz for audio). Both streams are perfectly temporally aligned.

🎛️ Multimodal Guidance System

LTX-2 pipelines use multimodal guidance to steer the diffusion process for both video and audio modalities. Each modality has independent guiders with separate parameters, enabling fine-grained control.

📊 Guidance Parameters Explained

cfg_scale float, default: 3.0-7.0

Classifier-Free Guidance strength. Higher values make output adhere more to text prompt. Typical range: 2.0-5.0 for video, 5.0-7.0 for audio. Set to 1.0 to disable CFG.

stg_scale float, default: 1.0

Spatio-Temporal Guidance strength. Improves temporal coherence by adding perturbation-based guidance. Typical values: 0.5-1.5. Set to 0.0 to disable STG.

stg_blocks list, default: [29]

Which transformer blocks to perturb for STG (0-47, where 47 is final block). Example: [29] targets final video+audio block. Set to [] to disable STG.

rescale_scale float, default: 0.7

Rescale factor for guided prediction variance matching. Prevents over-saturation when guidance is too strong. Typical: 0.5-0.7. Set to 0.0 to disable.

modality_scale float, default: 3.0

Modality CFG for audio-visual sync. Steers away from unsynced A/V results. For joint A/V generation: 3.0+. For video-only: set to 1.0.

skip_step int, default: 0

Skip guidance every N steps. Can speed up inference with minimal quality loss. Example: 2 applies guidance every other step. Set to 0 to disable (never skip).

⚙️ Example Guidance Configuration

from ltx_core.components.guiders import MultiModalGuiderParams # Video guider: moderate CFG, STG enabled, modality isolation video_guider_params = MultiModalGuiderParams( cfg_scale=3.0, # Moderate text adherence stg_scale=1.0, # Temporal coherence enabled rescale_scale=0.7, # Prevent over-saturation modality_scale=3.0, # Audio-visual sync stg_blocks=[29], # Target final block ) # Audio guider: higher CFG for stronger pronunciation adherence audio_guider_params = MultiModalGuiderParams( cfg_scale=7.0, # Strong text adherence for speech stg_scale=1.0, # Temporal coherence rescale_scale=0.7, modality_scale=3.0, # Audio-visual sync stg_blocks=[29], ) # Usage in pipeline pipeline( prompt="A person speaking clearly", video_guider_params=video_guider_params, audio_guider_params=audio_guider_params, )

💡 Pro Tips:

• Start with default values from constants.py and adjust based on results

• Higher cfg_scale = stronger prompt adherence but potentially less natural motion

• Higher stg_scale = better temporal coherence but slower inference (extra forward passes)

• For A/V generation, always set modality_scale > 1.0 (e.g., 3.0) for better lip-sync

• For video-only generation, set modality_scale = 1.0 to disable sync guidance

📋 Pipeline-Specific Steps

TI2VidTwoStagesPipeline Flow

1

Load Model & Components

Initialize LTX transformer, video VAE, audio VAE, text encoder (Gemma), vocoder, and spatial upsampler from checkpoint files.

2

Stage 1: Generate Low-Res with Multimodal Guidance

Run ~40 denoising steps on low-resolution latents with full CFG, STG, and modality guidance enabled.

3

Stage 2: Spatial Upsampling + Distilled LoRA Refinement

Upsample Stage 1 output, apply distilled LoRA weights, run ~20 denoising steps for high-resolution output.

4

Decode & Export

Decode high-res latents through VAE and vocoder. Export synchronized MP4 with video at specified height/width and audio at 24 kHz.

ICLoraPipeline (Video-to-Video) Flow

1

Encode Reference Video

Extract keyframes from reference video at specified intervals. Encode each frame through video VAE to latent space.

2

Stage 1: IC-LoRA Conditioned Generation

Initialize noise, apply IC-LoRA conditioning, run denoising with reference video guidance and multimodal CFG.

3

Stage 2: Upsampling

Upsample Stage 1 output with distilled LoRA refinement for higher resolution video-to-video transformation.

KeyframeInterpolationPipeline Flow

1

Encode Keyframes

Encode each keyframe image through video VAE to latent space, storing as additive guiding signals.

2

Stage 1: Generate Interpolation

Initialize noise, apply additive guiding signals from keyframes, run denoising for smooth interpolated frames.

3

Stage 2: Upsampling

Upsample interpolated latents with distilled LoRA for final high-resolution animation.

⚡ Optimization Tips

🔻 Reduce Memory Footprint with FP8:

Enable FP8 quantization for transformer with: --enable-fp8 flag + environment variable PYTORCH_CUDA_ALLOC_CONF=expandable_segments:True

⚡ Speed Up Denoising:

Use gradient estimation denoising loop for fewer steps (~20-30 vs 40) while maintaining quality. Available in ltx_pipelines.utils.helpers.

⏭️ Skip Guidance Steps:

Set skip_step=2 in guider params to apply guidance every other step, reducing inference time with minimal quality loss.

💾 Memory Cleanup Between Stages:

Pipelines automatically clean GPU memory between stages. Can be disabled if sufficient VRAM available for faster runtime.

🎬 Choose Right Pipeline:

Use DistilledPipeline for fastest inference (8 steps), TI2VidTwoStagesPipeline for best quality, or TI2VidOneStagePipeline for prototyping.

🖥️ Command-Line Usage

All pipelines can be run directly from command line:

# Run two-stage text-to-video pipeline python -m ltx_pipelines.ti2vid_two_stages \ --checkpoint-path /path/to/checkpoint.safetensors \ --distilled-lora /path/to/distilled_lora.safetensors 0.8 \ --spatial-upsampler-path /path/to/upsampler.safetensors \ --gemma-root /path/to/gemma \ --prompt "A beautiful sunset over the ocean" \ --output-path output.mp4 \ --height 512 \ --width 768 \ --num-frames 121 \ --num-inference-steps 40 # View all options for any pipeline python -m ltx_pipelines.ti2vid_two_stages --help

Select pipeline based on quality vs speed trade-off and conditioning type needed.

Adjust guidance parameters for optimal results - higher CFG for adherence, STG for coherence.

⚡ Optimize Resources

Use FP8, gradient estimation, and skip steps to reduce memory and inference time.

🎬 Audio-Visual Sync

Always set modality_scale > 1.0 for joint A/V generation to ensure lip-sync quality.
